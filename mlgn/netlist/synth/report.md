## 1. What was exported

`python -m mlgn.netlist.synth.export_fsm` rebuilds the checkpoint
(`mlgn/seqlgn/results/ckpt_cp50A_curr_c35.pt`, mechanism *combo*, task copy,
alphabet 8, hidden 1024, discrete test_acc 1.0) via the RNG-replay path
(`extract.rebuild_model`; accuracy gate re-passed at 1.0000), extracts the netlist
with `ir.extract_netlist` — **9 PIs, 1024 latches, 7168 two-input gates** (all 16
difflogic ops occur) — and emits:

| artifact | contents |
|---|---|
| `fsm.v` (265 KB, 9,248 lines) | synthesizable Verilog-2001, module `lgn_fsm`: `clk`, synchronous `rst` (loads `net.init` = all-zeros), `x[8:0]`, `output reg [1023:0] q`; one `assign` per gate with `ir.GATE_FN` semantics; internal names match `blif.signal_name` (`x<i>`/`q<i>`/`g<i>`/`c0`/`c1`) |
| `fsm.blif` (258 KB) | the same netlist via `blif.emit_blif`, with **all 1024 `q` signals as outputs** (`dataclasses.replace(net, outputs=[q0..q1023])`) so synthesis can sweep nothing |
| `fsm_clk.blif` | `fsm.blif` with `.latch ... re clk ...` (explicit clock, + `clk` PI), because yosys turns the 3-token ABC latch form into clockless `$_FF_` cells (§4); generated by `export_fsm.derive_clk_blif` |
| `golden_sym0..7.txt` | 40 frames × 1024-bit state as 256-hex-digit lines per legal write (cue+one-hot at t=0, then blanks), dumped from the bit-exact python simulator `sim.py` |
| `tb_fsm.v`, `ops_test.v`, `tb_ops.v` | testbenches (§2) |

| `fsm_synth.v` | post-`synth_xilinx` netlist written back for gate-level re-simulation (§3) |

Reproduce the WSL half with `bash run_all.sh` from this directory (runs both
testbenches, all four yosys flows, writes `fsm_synth.v`, and re-simulates it).
Raw logs: `log/`.

## 2. Simulation equivalence (iverilog) — PASS 8/8

Two levels, both against truth generated by the python simulator that is itself
bit-exact vs the torch model (9.2M trajectory bits, established previously):

1. **Gate-op micro-test** (`ops_test.v` + `tb_ops.v`): a 16-gate netlist, one gate
   per op on free inputs, exhaustively checked on all 4 input combinations against
   `ir.GATE_FN` — specifically guarding the asymmetric ops 2 (`a&~b`), 4 (`~a&b`),
   11 (`a|~b`), 13 (`~a|b`). **Result: `OPS: 4/4 cases PASS`.**
2. **Golden-vector FSM equivalence** (`fsm.v` + `tb_fsm.v`): for each of the 8
   legal writes, the TB resets, drives cue+one-hot at t=0 then blanks, and compares
   the full 1024-bit `q` against the golden file **cycle-exactly for 40 frames**
   (327,680 state bits total; the 40-frame window covers the whole ≤15-step
   transient plus 25 settled frames). **Result:**

   ```
   EQUIV: 8/8 symbols PASS, 0 frame mismatches
   RESULT: PASS
   ```

   (compile <1 s, sim ~3 s; log: `log/iverilog_equiv.log`)
3. **Random-stimulus probe** (`rand_probe.py` → `tb_rand.v`, added in adversarial
   review): 6 sequences × 30 frames of *fully random* 9-bit inputs each cycle —
   multi-hot symbols, cue toggling mid-sequence, patterns the copy protocol never
   reaches. Expected values come through an independent path: `fsm.blif` is parsed
   back into an `ir.Netlist` (covers → ops) and simulated with `sim.py` (the parsed
   netlist first replays `golden_sym0.txt` exactly as a sanity anchor).
   **Result: `RAND: 6 seqs x 30 frames PASS`** — on both `fsm.v` and the
   post-synthesis `fsm_synth.v` (§3), so the register optimization is exercised
   beyond protocol-reachable states.

The review also re-derived every gate semantically: an independent parser turned
all 7,168 `fsm.v` gate expressions and all 7,168 `fsm.blif` covers into truth
tables and compared them exhaustively (plus all 1,024 latch next-state names and
the init vector) — 0 mismatches. A second, independent review pass re-ran the
same cross-check with its own parser/evaluator (again 0 mismatches; op histogram
0..15 = 169/1234/199/484/1220/420/101/1274/245/120/430/231/399/215/209/218) and
mutation-tested the TB: flipping asymmetric op 2→4 on a trajectory-observable
gate (`g46`) makes `tb_fsm.v` FAIL as required, while the same flip on a masked
gate (`g7`) passes — i.e., the golden/random TBs detect observable gate errors
but, like any simulation, cannot see logic that is unobservable under the driven
stimuli (the per-gate truth-table cross-check covers those exhaustively).

### Flow C — generic: `read_verilog fsm.v; synth -top lgn_fsm; stat` (10.4 s)

3,138 cells = **885 FFs** (884 `$_SDFF_PP0_` + 1 `$_DFF_P_`) + **2,253 combinational
gates**: 696 MUX, 328 AND, 305 ANDNOT, 184 OR, 172 NAND, 162 NOT, 145 ORNOT,
144 NOR, 84 XNOR, 33 XOR. (7,168 raw IR gates → 2,253 after optimization, 3.2×.)

### Why 885 / 1,019 FFs and not 1,024?

All 1,024 state bits remain visible (1,024 OBUF-driven outputs in every flow, and
`CHECK` reports 0 problems in flows A/B), but yosys legally merges/folds registers.
A structural const-prop + hash-cons analysis of the IR (mirroring `opt_expr`/
`opt_merge`) finds 701 of the 7,168 gate outputs are structural constants yet only
**1** register is *structurally* redundant; the further reduction to 885 in flow A
comes from ABC's *semantic* optimization enabling iterated FF merging (the
BLIF flow's different starting structure reaches only 1,019). Both flows are
behavior-preserving — flow A's output re-passes the full golden suite (§3).

## 6. Honest caveats

- **No place-and-route, no timing.** These are post-synthesis cell counts only; no
  Fmax, congestion, or power numbers — that needs nextpnr-xilinx or Vivado next.
  (Depth is small: the IR is ~10 logic levels, so triple-digit MHz is plausible but
  *unmeasured*.)
- **I/O-bound as exported:** 1,035 port bits (1,024 `q` + 9 `x` + clk/rst) exceed
  any Artix-7 package — the full-state port exists to make verification airtight,
  not for deployment. A deployable top needs the GroupSum/argmax head in fabric
  (popcount trees over 8×128 bits; Agent HEAD's in-netlist head is the natural
  source), shrinking outputs to 3–11 bits.
- **Reset/init differ by flow:** `fsm.v` uses an explicit synchronous `rst`;
  the BLIF flows carry latch init values (GSR-style init attributes, no reset
  port). Functionally equivalent for the protocol (init = all-zeros) but not
  literally identical netlist interfaces.
- **The BLIF cross-check was run on `fsm_clk.blif`** (mechanical `re clk`
  rewrite of `fsm.blif`); the emitted `fsm.blif` itself stays in the ABC-native
  3-token form the model checker consumes.
- Equivalence evidence is simulation (protocol golden vectors + random stimulus)
  plus a gate-by-gate semantic cross-check of `fsm.v` against `fsm.blif` — not a
  formal RTL-vs-IR sequential-equivalence proof. The formal statements about the
  state machine live in the ABC theorems on the *same* IR; a `yosys equiv`/SEC
  pass RTL-vs-BLIF is future work. (Coverage note: the per-gate truth-table
  cross-check *is* exhaustive at the combinational level; only the composed
  sequential behavior relies on simulation.)
