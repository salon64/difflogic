# netlist — checkpoint → gate-level netlist → formal verification

The **shared exporter artifact** identified by the program validation
(research/20 §E.4): the single missing piece that simultaneously fixes P2's beat-4
verification demo, de-risks P3a, re-arms P3b's hardware timestamp, and backs the
Kyushu pitch. This folder contains its v0 plus the **P3a one-afternoon falsifier**
(research/20 §D1): export a trained disc=1.000 copy checkpoint and model-check a
hold invariant with ABC (`pdr` / `bmc3`).

## Why this is possible at all

At eval, EVERY seqlgn mechanism is an exact-binary deterministic clocked FSM:
`model.eval()` argmaxes each neuron to one of the 16 gates, inputs are binarised, and
the state is a {0,1} register file. `gated`/`combo`/`clatch` all deploy to the same
circuit shape — candidate MLP + gate MLP + one 3-gate MUX per state bit + registers
(combo's state-round and clatch's enable-round are identity on binary values).

## The wiring problem (and the replay)

`LogicLayer.indices` is generated by two `torch.randperm` calls on the **global CPU
generator** at construction and is NOT in the state_dict — a checkpoint alone cannot
reconstruct the circuit (research/20 footgun #2). But for every run in
`run_queue.sh` the wiring is exactly replayable:

* `train.py` seeds (`utils.set_seed`) and builds the task **before** the model, and
  every task builder consumes **zero** draws from the global CPU stream (synthetic
  tasks use a dedicated `torch.Generator`; MNIST uses dedicated generators for the
  permutation/split; DataLoaders only draw at iteration time).
* On CUDA, each LogicLayer's weight-init `randn` goes to the **CUDA** stream; only
  the two `randperm`s touch the CPU stream — in module-registration order.

So `extract.rebuild_model` re-seeds and regenerates each layer's two randperms in
`model.modules()` order. Correctness is **gated, never assumed**:

1. the rebuilt model must reproduce the run's recorded discrete `test_acc` on the
   regenerated (bit-identical) test set — wrong wiring ⇒ ~chance accuracy;
2. the extracted netlist must match the torch model **bit-for-bit** (per-timestep
   state trajectories + predictions) on real data (`sim.equivalence_check`).

(Forward fix — registering `indices` as persistent buffers — intentionally NOT in
v0: it changes the checkpoint format and touches upstream `difflogic/difflogic.py`;
do it as its own change once the falsifier verdict is in.)

## Files

| file | role |
|---|---|
| `extract.py` | RunSpec from a results JSON (+ `--alphabet` etc. from run_queue.sh, which the JSONs don't record); RNG-replay rebuild; accuracy gate |
| `ir.py` | flat topological gate IR (16-op vocabulary, layer slices, latches); `extract_netlist(model)`; remap-replay helper for property construction |
| `sim.py` | vectorized numpy simulator; bit-exact equivalence vs torch; copy-protocol trajectory analysis (settle time + decode) |
| `props.py` | property miters, one `bad` output each (see below) |
| `blif.py` | BLIF emission (`.names` covers per gate, `.latch` per state bit) for ABC |
| `falsify.py` | CLI: end-to-end run → `out/<ckpt>/report.json` + BLIFs + ABC logs |

## Properties

| name | question | engine |
|---|---|---|
| `comb_hold` | ∀ states h (reachable or not): does a blank input hold h exactly? | `sat` (UNSAT = yes) |
| `seq_hold` | on all reachable states from reset, with free inputs: blank ⇒ hold? | `pdr` / `bmc3` |
| `protocol_hold` | copy protocol (legal cue+one-hot write at t=0, forced blanks after): does the state stop changing within K steps of the write, **forever**? | `pdr` / `bmc3` |
| `protocol_hold_anyx0` | same, but after ANY first input incl. garbage (robustness bonus) | `pdr` / `bmc3` |

The **settle window K** exists because of the first mechanistic finding (below).
`protocol_hold` PROVED + the 8 simulated write trajectories decoding correctly at
their fixed points = a complete functional-correctness theorem for the deployed
circuit at **arbitrary** delay ≥ K — strictly beyond any bounded testing.

## Findings (2026-07-10, ckpt_cp50A_curr_c35 = combo copy-35, disc test_acc 1.0)

1. **The RNG replay reconstructs CUDA-trained checkpoints on a CPU host, exactly.**
   Rebuilt discrete accuracy 1.0000 == recorded (full 10k test set); netlist
   bit-exact vs torch over 9.2M state bits (this also survived the torch-version
   change DUST→local).
2. **The trained "register" does not latch immediately — it is an attractor.**
   After the write step, the state keeps changing for **9–15 further blank steps**
   before reaching a per-symbol fixed point (which then decodes correctly, 8/8).
   During the transient the decode is WRONG (defaults to class 2, wanders). So the
   deployed circuit has a minimum-delay operational envelope: correct for readout
   ≥ ~15 steps after the write, wrong below it — invisible to the training-length
   test set (L=35 readout), surfaced by the exporter on day one.
3. **Exhaustive closed-system analysis is itself a proof engine here.** After the
   write the inputs are forced blank, so the system is deterministic and closed:
   batched simulation of ALL 2^9 = 512 first inputs is a complete case analysis.
   Result: every input settles (legal ≤ 15 steps, garbage ≤ 18, zero limit
   cycles); the slowest are multi-hot "conflicting write" patterns (e.g. symbols
   {1,7} together). This fixes the honest arming delays K=16 (legal) / K=19 (any).
4. **Naive model checking FAILS at this size — the right recipe proves in seconds.**
   On the true K=16 hold invariant (1,041 latches / 8,183 ANDs after strash):
   - `pdr`: **timeout** 900 s, stuck at frame 16 (the arming frontier);
   - `ind` (k-induction): **timeout** 300 s at frame 17 (204k conflicts);
   - `bmc3`: clean to 310 frames in 240 s (bounded evidence only);
   - **`tempor -F 20; scorr; dc2; pdr`: PROVED in ~2 s** — temporal decomposition
     unrolls past the (simulation-derived) transient, inductive signal
     correspondence then collapses the settled cone to constant-false.
   Same recipe proves the K=19 any-input variant in ~3 s (`tempor -F 21` alone
   collapses it). **Recipe: settle bound by simulation → `tempor` past it →
   `scorr` → `pdr`.** This is the method delta beyond push-button MC.
5. **Two machine-checked theorems about the deployed circuit** (ABC, end-to-end):
   - *Functional correctness at ANY delay*: for every legal write (cue + one-hot
     symbol), the state is fixed forever after 16 blank steps (`protocol_hold`
     PROVED) and its fixed point decodes to the written symbol (8/8 simulated
     trajectories) ⇒ the circuit provably solves copy for **arbitrary** delay
     ≥ 16 — strictly beyond any bounded test (bmc3 alone had only reached 310).
   - *Stability under arbitrary writes*: after ANY first input, including garbage
     the net never saw, the state converges within 19 steps and never moves again
     (`protocol_hold_anyx0` PROVED) — no input can trigger a non-converging orbit.
6. `comb_hold` is **SAT in 0.01 s** (7,998 ANDs / 18 levels): consistent with (2)
   — transient states are witnesses, so hold is a *reachability* property, not a
   combinational one. `seq_hold` (free inputs every step) CEXes at frame 0: even
   the reset state is not blank-stable. Both are diagnostics, not failures.

## The contrast case (ckpt_cpB_gated_oracle = gated copy-50, disc test_acc 0.3803)

Same pipeline, second mechanism/checkpoint — the replay gate reproduced the
recorded 0.3803 EXACTLY on the full test set (bit-exact netlist over 13.1M
trajectory bits at L=50), so the reconstruction is validated across mechanisms,
sequence lengths, and accuracy regimes. And it **explains the accuracy
mechanistically**:

- The gated circuit is ALSO provably stable: all 8 legal writes settle (9–21
  steps), all 512 first inputs settle by 21, zero limit cycles; both hold theorems
  (K=22) PROVED by the same recipe in < 1 s, while naive `pdr` again times out at
  exactly the arming frontier (frame 22).
- But only **3/8 fixed points decode correctly** (symbols 0, 2, 6) — and
  3/8 = 0.375 ≈ the measured 0.3803. The "38% accuracy" is not statistical: the
  deployed circuit deterministically maps 3 symbols to correct attractors and 5 to
  wrong ones.

So **hold + decode splits stability from correctness**: stability (settle-then-
hold-forever) is a property of the MUX-register architecture that both mechanisms
learn; correctness lives in which fixed points the write dynamics select. This is
the quantitative cross-mechanism contrast P3a can build tables around — and it is
invisible to soft-accuracy numbers (both checkpoints have test_soft ≈ 0.88).

## What this means for P3a (falsifier verdict)

The report's gate (research/20 §D1) asked: does export → ABC on a hold invariant
de-risk or reframe the paper? **Answer: de-risked, with a reframe.** The pipeline
works end-to-end and cheaply; but for cue-then-closed tasks (copy), the "model
checking" largely reduces to exhaustive simulation + temporal decomposition — IC3
contributes nothing until the inputs stay free at verification time. The genuine
MC territory (and the paper's core) is therefore **free-input properties**:
distcopy's "no distractor placement can corrupt the register" (exponential input
space, needs the popcount head), noise-tolerance envelopes, and the
cross-mechanism stability-vs-correctness contrast above.

Verdict tables and raw ABC logs: `out/ckpt_cp50A_curr_c35/` (primary, incl. the
`ind`/`scorr`/`tempor` ablation logs), `out/ckpt_cp50A_curr_c35_anyx0K19/` (K=19
naive-engine record: pdr UNDECIDED @900 s, bmc3 clean to 124 frames, recipe PROVED
in 3 s), `out/ckpt_cpB_gated_oracle/` (contrast run).
